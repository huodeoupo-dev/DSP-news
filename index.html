# 📚 DSPNews 每日自动更新 - 超级详细图文教程 > **目标**：让网站每天早上10点自动抓取最新音乐行业新闻 --- ## 📋 完整流程概览 ``` 第一步：在 GitHub 创建账号并创建仓库 ↓ 第二步：下载代码并上传到你的 GitHub ↓ 第三步：在 GitHub Actions 启用自动更新 ↓ 第四步：部署网站到 Vercel ↓ ✅ 完成！每天10点自动更新 ``` --- ## 第一步：在 GitHub 创建账号并创建仓库 ### 1.1 打开 GitHub 网站 在浏览器中打开（推荐使用 Chrome 浏览器）： ``` https://github.com ``` 你会看到这样的页面： ``` ┌─────────────────────────────────────────────────────────────┐ │ │ │ GitHub │ │ Where the world builds software │ │ │ │ [Sign up] [Sign in] │ │ │ └─────────────────────────────────────────────────────────────┘ ``` ### 1.2 注册 GitHub 账号 1. 点击 **Sign up**（注册） 2. 按照提示填写信息： - 📧 邮箱地址（建议使用 Gmail 或 QQ 邮箱） - 🔐 密码（至少8位，建议包含字母和数字） - ✅ 用户名（这是你的唯一标识，建议用英文，比如 `myname`） 3. 验证你的邮箱（GitHub 会发一封验证邮件到你的邮箱，点击邮件中的链接即可） ### 1.3 创建新的代码仓库 1. 登录后，点击右上角的 **+** 号 2. 选择 **New repository**（新建仓库） ``` ┌─────────────────────────────────────────────────────────────┐ │ + ▼ Your repositories │ │ ───────────────────────── │ │ New repository │ │ Import repository │ └─────────────────────────────────────────────────────────────┘ ``` 3. 填写仓库信息： ``` ┌─────────────────────────────────────────────────────────────┐ │ │ │ Repository name: (填写这个) │ │ ┌───────────────────────────────────────────────────────┐ │ │ │ dsp-news-dashboard │ │ │ └───────────────────────────────────────────────────────┘ │ │ │ │ Description (optional): │ │ ┌───────────────────────────────────────────────────────┐ │ │ │ 音乐行业新闻每日自动更新仪表板 │ │ │ └───────────────────────────────────────────────────────┘ │ │ │ │ ○ Public ← 选择这个（推荐） │ │ ● Private │ │ │ │ ☑ Add a README file │ │ │ │ [Create repository] ← 点击这个按钮 │ └─────────────────────────────────────────────────────────────┘ ``` **恭喜你！仓库创建成功！** 🎉 --- ## 第二步：下载并上传代码 ### 2.1 下载代码包 我已经为你准备好了完整的代码包，请联系我来获取下载链接。 ### 2.2 解压代码包 1. 找到下载的文件（通常在 **下载** 文件夹） 2. 右键点击文件 3. 选择 **解压到当前文件夹**（Windows）或 **双击解压**（Mac） 解压后会看到一个文件夹叫 `dsp-news-dashboard` ### 2.3 上传代码到 GitHub #### 方法一：使用网页上传（最简单） 1. 打开你刚才创建的 GitHub 仓库页面 2. 点击 **uploading an existing file** 链接 ``` ┌─────────────────────────────────────────────────────────────┐ │ │ │ dsp-news-dashboard │ │ ┌───────────────────────────────────────────────────────┐ │ │ │ 👆 dragging and dropping files to upload │ │ │ │ directly to the main branch │ │ │ │ │ │ │ │ or updating an existing file? │ │ │ │ use the file editor │ │ │ │ │ │ │ │ uploading an existing file ← 点击这里 │ │ │ └───────────────────────────────────────────────────────┘ │ │ │ └─────────────────────────────────────────────────────────────┘ ``` 3. 打开解压后的 `dsp-news-dashboard` 文件夹 - **全选** 所有文件和文件夹（Ctrl+A 或 Command+A） - **拖拽** 到 GitHub 网页上 4. 确认上传： - 滑动到页面底部 - 点击 **Commit changes**（提交更改） #### 方法二：使用 GitHub Desktop（推荐，更稳定） 1. 下载 GitHub Desktop： ``` https://desktop.github.com ``` 2. 安装并登录 3. 点击 **Clone a repository**（克隆仓库） 4. 选择你刚创建的 `dsp-news-dashboard` 5. 将解压的代码复制到克隆的文件夹中 6. GitHub Desktop 会自动检测到更改 7. 填写摘要，点击 **Commit to main** 8. 点击 **Push origin**（推送到云端） --- ## 第三步：启用自动更新（GitHub Actions） ### 3.1 打开 Actions 页面 1. 在你的仓库页面，点击顶部的 **Actions** 标签 ``` ┌─────────────────────────────────────────────────────────────┐ │ <> Code ○ Issues pull requests Actions Projects │ │ ↑ │ │ 点击这里 │ └─────────────────────────────────────────────────────────────┘ ``` ### 3.2 启用 GitHub Actions 如果看到提示，点击 **Enable Actions**： ``` ┌─────────────────────────────────────────────────────────────┐ │ │ │ Get started with GitHub Actions │ │ │ │ [Enable Actions] ← 点击这个按钮 │ │ │ └─────────────────────────────────────────────────────────────┘ ``` ### 3.3 查看每日更新工作流 你会在左侧看到 **Daily News Update** 工作流： ``` ┌─────────────────────────────────────────────────────────────┐ │ Workflows │ │ ───────────────── │ │ ✓ Daily News Update ← 应该看到这个 │ │ All workflows │ └─────────────────────────────────────────────────────────────┘ ``` ### 3.4 手动触发首次更新（可选） 1. 点击左侧的 **Daily News Update** 2. 点击右侧的 **Run workflow** 3. 点击绿色的 **Run workflow** 按钮 4. 等待几秒钟，你会看到工作流开始运行 5. 显示 ✅ 绿色勾号表示成功！ --- ## 第四步：部署网站到 Vercel ### 4.1 注册 Vercel 账号 1. 打开网站： ``` https://vercel.com ``` 2. 点击 **Sign Up**（注册） 3. 可以用 **GitHub 账号登录**（最简单！） - 点击 "Continue with GitHub" - 授权确认即可 ### 4.2 导入 GitHub 仓库 1. 登录后，点击 **Add New** → **Project** 2. 找到你的 `dsp-news-dashboard` 仓库 - 可能需要点击 "Install Vercel for GitHub" - 按照提示授权 3. 点击仓库旁边的 **Import**（导入） ### 4.3 配置并部署 在配置页面： ``` ┌─────────────────────────────────────────────────────────────┐ │ │ │ Configure Project │ │ │ │ Framework Preset: [Vite] ← 自动检测到 │ │ Root Directory: [./] │ │ Build Command: [pnpm build] │ │ Output Directory: [dist] │ │ │ │ Environment Variables: (可以留空) │ │ │ │ [Deploy] ← 点击这个按钮 │ └─────────────────────────────────────────────────────────────┘ ``` ### 4.4 等待部署完成 1. 你会看到部署进度 2. 显示 ✅ **Done** 表示成功！ 3. 你会看到一个链接，比如： ``` https://dsp-news-dashboard.vercel.app ``` **这就是你的网站链接！** 🎉 --- ## 🎯 完成！ ### 验证设置是否成功 1. 打开你的 Vercel 网站链接 2. 你应该能看到新闻仪表板 3. 点击 **Actions** 标签，确认看到绿色勾号 ### 自动更新原理 ``` 每天早上 10:00（北京时间） ↓ GitHub Actions 自动运行 ↓ 抓取最新新闻 ↓ 更新网站数据 ↓ Vercel 自动部署 ↓ 你的网站显示最新内容！ ``` --- ## ❓ 常见问题解答 ### Q: 我没有 GitHub 账号怎么办？ **A:** 访问 https://github.com 点击 Sign up，按照提示注册即可，完全免费！ ### Q: 上传文件失败怎么办？ **A:** -
